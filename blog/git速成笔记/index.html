Git

git是一款分布式版本控制系统，是由Linus编写的，由此诞生了GitHub网站（2008年上线）

Git的安装

安装包下载：https://github.com/gohugoio/hugo/releases

安装完成后需要在Git bash中登录GitHub账号

$ git config --global usr.name "XXX"
$ git config --blobal usr.email "email"

git init：在当前目录创建可管理的仓库

Git功能与常用方法

Git功能可分为以下四个方面：

  1. 基本的文件管理
  2. 远程仓库
  3. 分支管理
  4. 标签管理

基本的文件管理

git add filename 添加文件

git commit -m "message" 提交add的文件，并附带提交说明

git status 查看工作区的状态（检查是否有未提交的文件）

git diff 查看工程中的历史修改内容

git log 显示提交日志

git reset --hard HEAD^ 回退到工程的上一版本（HEAD~100 回退100次）

  • --hard XXXXX 也可直接回退到某一版本号
  • Git回退版本的原理：指针
  • git reset HEAD filename 把暂存区的修改撤销掉，回退到工作区

git reflog 查看自己的每次命令（有所有命令对应的版本号）

git checkout -- filename 丢弃工作区的修改（保持上一版本的内容）

git rm filename 删除文件，此后应用 git commit 提交一次

远程仓库

  • 关联远程库
    • git remote add origin git@server-name:path/repo-name.git 关联远程库
    • git push -u origin master 第一次推送master分支的内容
    • git push origin master 推送最新修改
  • 克隆远程库
    • git clone git@server-name:path/repo-name.git 克隆远程库

分支管理

查看分支：git branch

查看远程库信息：git remote -v

创建分支：git branch name

在本地创建和远程分支对应的分支：git checkout -b branch_name origin/branch_name

切换分支：git checkout name or git switch name

创建+切换分支：git checkout -b name or git switch name

合并某分支到当前分支：git merge name or git merge --no-ff -m "message" name

删除分支：git branch -d name

查看分支合并冲突：git log --graph

临时保存：git stash

返回临时保存工作区：git stash pop

将某次修改复制到当前分支：git cherry-pick commit_id

丢弃没有合并过的分支：git branch -D name

  • 多人协作工作模式
    • 推送自己的修改：git push origin branch_name
      • 若推送失败，需试图合并：git pull
        • 若提示no tracking information 则应创建本地分支与远程分支的链接关系：`git branch –set-upstream-to branch_name prigin/branch_name
        • 若合并有冲突，则解决冲突，并在本地提交
      • 解决冲突或无冲突时，再推送git push origin branch_name

标签管理

新建标签：git tag tag_name

指定标签信息：git tag -a tag_name -m "message"

查看所有标签：git tag

推送本地标签：git push origin tag_name

推送所有未推送过的本地标签：git push origin tag_name

删除本地标签：git tag -d tag_name

删除远程标签：git push origin :refs/tag/tag_name
