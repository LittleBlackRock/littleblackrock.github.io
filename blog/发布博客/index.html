发布博客

1. 创建.md文件，编辑blog内容

hugo new blog/newBlog.md

2. 生成静态页面

hugo --theme=npq-hugo --baseUrl="https://littleblackrock.github.io/" --buildDrafts

3. 发布到GitHub

cd public
git add .
git commit -m "new blog added"
git push origin master

发布博客中采过的坑

1. 执行git push origin master时弹出以下错误信息

To https://github.com/LittleBlackRock/littleblackrock.github.io.git
 ! [rejected]        master -> master (fetch first)
error: failed to push some refs to 'https://github.com/LittleBlackRock/littleblackrock.github.io.git'
hint: Updates were rejected because the remote contains work that you do
hint: not have locally. This is usually caused by another repository pushing
hint: to the same ref. You may want to first integrate the remote changes
hint: (e.g., 'git pull ...') before pushing again.
hint: See the 'Note about fast-forwards' in 'git push --help' for details.

原因：远程仓库的代码与本地版本并不一致

解决：应在此之前先同步本地库和远程库的内容，即

cd public
git add .
git commit -m "new blog added"
git pull origin master
git push origin master
